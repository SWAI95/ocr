# 내부망(오프라인) 배포 가이드 — 한국어 문서 OCR 엔진

> 대상 환경: **Ubuntu 24.04.x LTS · Python 3.12 · RTX 4090 ×2 (24GB) · CUDA 드라이버 12.9.86**
> **OCR 전용 깨끗한 Python 3.12 venv를 새로 만들고**, 그 위에 OCR 실행에 필요한 모듈을
> 전부(torch·paddle 포함) 설치한다.
> **외부 API·인터넷 일절 사용 불가** → 모든 휠·모델·Ollama 바이너리를 빌드 머신에서 받아 반입한다.

이 가이드는 두 머신을 가정한다.

| 머신 | 인터넷 | 역할 |
|---|---|---|
| **빌드 머신** | ✅ O | 휠/모델/Ollama 번들 수집 (개발 5090 머신 또는 임시 온라인 PC) |
| **내부망 머신** | ❌ X | 실제 배포 대상 (4090 ×2) |

> ⚠️ **빌드 머신과 내부망 머신은 동일 베이스여야 한다**: `linux x86_64`, **Python 3.12**.
> 휠은 파이썬 minor 버전에 묶인다(`cp312`). 빌드 머신이 3.12, 내부망이 3.10이면
> 휠이 안 맞아 설치가 깨진다. **양쪽 다 3.12**로 통일한다.

## 🔑 환경 결정 — 왜 "새 venv"인가 (꼭 읽기)

기존 `aicore` 환경을 복사해 재활용하려던 방식은 **폐기**했다. 이유:

- **aicore venv는 Python 3.10**이다. **venv의 파이썬 버전은 생성 시점에 고정**되어,
  3.10 venv를 아무리 다시 복사해도 계속 3.10이다. 복사로는 3.12가 안 된다.
- 배포 타깃(24.04)과 빌드 머신이 모두 3.12이므로, **3.12로 새 venv를 만드는 것**이
  휠 호환(`cp312`)과 정합성 면에서 유일하게 깔끔한 길이다.
- 새 venv면 torch도 어차피 새로 깔지만 **cu129 휠로 설치하므로 동작은 동일**하다
  ("torch만 빼고 다 바꿔도 된다"는 요구 = torch는 cu129 버전 유지, 그대로 충족).

> ⚠️ **맨이름 `python` / `pip` 쓰지 말 것.** 이 환경에서 `python`은 3.10이다.
> 항상 `python3.12` 또는 **활성화한 venv의 `.venv/bin/python` / `.venv/bin/pip`** 만 쓴다.
> pip은 파이썬 버전마다 따로 존재한다 — `python3.12 -m pip --version` 끝에 `(python 3.12)`
> 가 찍히는지로 확인한다.

## 인터넷 될 때 / 안 될 때 — 단계 한눈에

이 문서는 **인터넷 되는 곳에서 "받기"(PART 1) → 반입 → 인터넷 안 되는 곳에서 "설치·실행"(PART 2)** 순서다.
**네트워크가 필요한 작업은 PART 1에서 전부 끝낸다.** PART 2는 단 한 번도 외부에 접속하지 않는다.

| 단계 | 인터넷 | 어디서 | 무엇을 |
|---|:---:|---|---|
| 1. 빌드 환경 확인 | ✅ 필요 | 빌드 머신 | Python 3.12 + pip 있는 환경 확보 |
| 2. requirements 확인 | ⬜ 무관 | 빌드 머신 | `internal_full.txt` (이미 레포에 있음) |
| 3. 휠 수집 → `wheelhouse/` | ✅ 필요 | 빌드 머신 | PyPI·PyTorch(cu129)·Paddle(cu129) + pip 부트스트랩 휠 |
| 4. 모델 수집 → `models/` | ✅ 필요 | 빌드 머신 | Paddle·EasyOCR·PaddleOCR-VL 가중치 |
| 5. 시스템 .deb → `debs/` | ✅ 필요 | 빌드 머신 | python3.12-venv·fonts-nanum(+선택 poppler/tesseract) |
| 6. Ollama 번들 → `ollama_bundle/` | ✅ 필요 | 빌드 머신 | ollama 바이너리 + qwen2.5vl 모델 |
| 7. 번들 구성 → 반입 | — | (이관) | 한 폴더로 묶어 USB/승인 매체로 반입 |
| 8. venv 생성 + 설치 | ❌ **불가** | 내부망 머신 | `python3.12 -m venv` + `--no-index` 오프라인 설치 |
| 9. Ollama 설치 | ❌ **불가** | 내부망 머신 | tarball 풀기 + 모델 blob 복원 |
| 10. 검증·실행 | ❌ **불가** | 내부망 머신 | `OCR_OFFLINE=1` 스모크/서버 |

> 💡 **빠른 점검**: PART 2(8~10단계)에서 `pip`이 `--no-index`, OCR 실행이 `OCR_OFFLINE=1`
> 이면 외부 접속이 코드/플래그 차원에서 막힌다. 네트워크 케이블을 뽑은 상태로 통과하면 OK.

## 0. 핵심 전략 (정책)

**torch 계열만 cu129 버전으로 고정, 그 외는 전부 최신. numpy 는 2.x.**
이 정책은 `requirements/internal_full.txt` 한 파일에 모두 담겨 있다(아래 2단계).

1. **torch/torchvision 고정**: `torch 2.8.0+cu129` / `torchvision 0.23.0+cu129`
   — 4090 GPU 동작이 검증된 버전.
2. **그 외는 최신**: `numpy>=2.1`, `opencv`, `pypdfium2`, `jiwer`, `pdf2image`,
   `python-Levenshtein` 및 전이 의존성(`scipy`/`pandas`/`scikit-image` 등)을 numpy 2 호환 최신으로.
3. **OCR 엔진 코어 3종은 검증된 버전으로 핀**: `paddleocr==3.6.0`, `paddlex[ocr]==3.6.1`,
   `transformers==4.57.6`. 엔진 코드가 이들 3.6.x API(`res.markdown` 프로퍼티,
   `PaddleOCRVL(vl_rec_backend="native")`, PPStructureV3 파라미터)에 결합돼 있어
   minor 업그레이드만으로도 인식이 깨질 수 있다.
4. CUDA 휠은 **cu129** 로 통일(4090은 cu126/128/129 모두 GPU 정상; torch가 cu129라 충돌 최소).
5. `paddlepaddle-gpu` 는 **cu129 휠**. (의존성 충돌이 나면 `--no-deps` 로 따로 설치 — 8단계 참고.)

> ✅ 새 venv이므로 numpy 1.x 잔재가 없다. 기존 환경 상향(eager) / 복사본 깨짐 같은
> 복잡성은 전부 사라진다 — `internal_full.txt` 하나를 깨끗한 3.12 venv에 설치하면 끝.

## 1. 빌드 머신: Python 3.12 + pip 환경 확보 (인터넷 O)

휠을 받으려면 **pip이 있는 Python 3.12** 가 필요하다. 둘 중 하나:

```bash
# 방법 A) 프로젝트의 기존 .venv 사용 (이미 3.12 + pip 완비면 가장 간단)
source .venv/bin/activate
python --version          # → Python 3.12.x
python -m pip --version   # → ... (python 3.12)

# 방법 B) 빌드 전용 깨끗한 3.12 venv 새로 (ensurepip 있는 경우)
python3.12 -m venv buildenv && source buildenv/bin/activate
python -m pip install -U pip setuptools wheel
```

> ⚠️ `python3.12 -m venv` 가 `No module named 'ensurepip'` 로 실패하면, 그 시스템엔
> `python3.12-venv` 패키지가 없는 것이다(빌드 머신은 인터넷이 되니
> `sudo apt-get install python3.12-venv` 로 해결). **내부망 머신에서는 이게 함정이 되므로
> 5단계에서 그 .deb 를 챙기고 8단계에서 pip 부트스트랩으로 우회한다.**

이후 PART 1의 모든 `pip` 은 **이 활성화된 3.12 환경의 pip** 을 가리킨다.

## 3. 빌드 머신: 파이썬 휠 수집 → `wheelhouse/`

cu129 인덱스에서 받는다. **세 인덱스**(PyPI + PyTorch cu129 + Paddle cu129)를 함께 건다.
`--only-binary=:all:` 로 **휠만** 받아(오프라인에서 컴파일이 필요 없게) 한다.

```bash
cd /path/to/ocr           # 프로젝트 루트
mkdir -p wheelhouse

# (a) closure — torch + OCR + numpy2 + web (paddlepaddle-gpu 는 internal_full 에서 제외됨)
python -m pip download -r requirements/internal_full.txt \
  -d wheelhouse --only-binary=:all: \
  --index-url https://pypi.org/simple \
  --extra-index-url https://download.pytorch.org/whl/cu129 \
  --extra-index-url https://www.paddlepaddle.org.cn/packages/stable/cu129/

# (b) ★필수★ paddlepaddle-gpu 는 반드시 단독 --no-deps 로.
#     (torch 와 nvidia-cu12-* 핀이 충돌해 (a)에 같이 넣으면 ResolutionImpossible)
python -m pip download "paddlepaddle-gpu==3.3.1" \
  -d wheelhouse --no-deps --only-binary=:all: \
  --index-url https://www.paddlepaddle.org.cn/packages/stable/cu129/

# (c) ★중요★ pip 부트스트랩용 휠도 함께 (내부망 venv가 ensurepip 없을 때 대비)
python -m pip download pip setuptools wheel -d wheelhouse --only-binary=:all:

echo "받은 휠: $(ls -1 wheelhouse | wc -l) 개 / $(du -sh wheelhouse | cut -f1)"
```

> 📌 **검증 2가지**:
> 1. `wheelhouse/` 에 `paddlepaddle_gpu-3.3.1-...cu129...whl` 가 있는지 눈으로 확인.
>    cu126/cpu 휠이 섞이면 4090에서 GPU가 안 잡힌다.
> 2. `ls wheelhouse/*.tar.gz` 로 **sdist(소스 배포)가 섞였는지** 확인. `--only-binary=:all:`
>    면 보통 없지만, 있으면 그 패키지는 오프라인에서 컴파일이 필요할 수 있다 →
>    빌드 도구까지 챙기거나 해당 패키지 버전을 휠 있는 것으로 조정.

> ⚠️ **(a)+(b) 분리는 선택이 아니라 필수다.** `paddlepaddle-gpu` 를 `internal_full.txt` 에
> 넣거나 (a)에 합치면 torch 와 CUDA 의존성 충돌로 resolve 가 실패한다(실측). 설치 단계(8-4)도
> 같은 이유로 paddle 을 `--no-deps` 로 따로 깐다.

### 4-1. Paddle(텍스트·레이아웃·표·도장) + EasyOCR 모델

```bash
python scripts/download_models.py          # → models/paddle/official_models, models/easyocr
#   --with-formula  : 수식/차트 모델까지(번들 커짐, 계약서엔 보통 불필요)
```

### 4-2. PaddleOCR-VL(0.9B) 모델 — paddle_vl / hybrid 의 오프라인 VLM

> ⚠️ `download_models.py` 는 PaddleOCR-VL을 받지 **않는다**. paddle_vl·hybrid(완전
> offline VLM 경로)를 쓰려면 아래를 **추가로 1회** 실행해 모델을 번들에 포함시킨다.

```bash
python - <<'PY'
import os
from pathlib import Path
root = Path.cwd() / "models"
os.environ["PADDLE_PDX_CACHE_HOME"] = str(root / "paddle")
os.environ["HF_HOME"] = str(root / "hf")
os.environ.setdefault("PADDLE_PDX_DISABLE_MODEL_SOURCE_CHECK", "True")
from paddleocr import PaddleOCRVL
# 인스턴스화만으로 PaddleOCR-VL-1.6-0.9B 가중치가 캐시로 다운로드된다.
PaddleOCRVL(vl_rec_backend="native", device="cpu",
            use_doc_orientation_classify=False, use_doc_unwarping=False)
print("[OK] PaddleOCR-VL 모델 다운로드 완료 →", root)
PY
```

받은 뒤 `models/paddle/` 와 `models/hf/` 크기를 확인한다(VL 모델 ~2GB 추가).

## 6. 빌드 머신: Ollama 바이너리 + Qwen2.5-VL 모델 수집

내부망엔 Ollama가 없으므로 **바이너리 tarball + 모델 blob**을 통째로 가져간다.
(hybrid 를 `paddle_vl` 백엔드로만 쓸 거면 Ollama 는 건너뛰어도 된다 — 11단계 참고.)

```bash
mkdir -p ollama_bundle

# (a) Ollama 리눅스 바이너리 (amd64) — ⚠️ 최신 릴리스는 .tar.zst (zstd) 포맷이다.
#     (예전 https://ollama.com/download/ollama-linux-amd64.tgz 는 404 — GitHub 릴리스에서 받는다.)
#     최신 버전 확인: https://github.com/ollama/ollama/releases/latest  (예: v0.30.11)
VER=v0.30.11
curl -fL --retry 3 \
  "https://github.com/ollama/ollama/releases/download/$VER/ollama-linux-amd64.tar.zst" \
  -o ollama_bundle/ollama-linux-amd64.tar.zst

# (b) Qwen2.5-VL 모델 blob 동봉
#     빌드 머신에 모델이 없으면 먼저: ollama pull qwen2.5vl:7b  (작게 가려면 :3b)
#     모델은 ~/.ollama/models 에 content-addressed 로 저장된다(MODELS 경로).
#     ★ 여러 모델이 섞여 있으면 qwen 만 골라 담는다(매니페스트가 가리키는 blob 만).
MODELS="$HOME/.ollama/models"     # Windows/WSL interop 면 /mnt/c/Users/<유저>/.ollama/models
MANIFEST="$MODELS/manifests/registry.ollama.ai/library/qwen2.5vl/7b"
{ echo "manifests/registry.ollama.ai/library/qwen2.5vl/7b"
  python3 - "$MANIFEST" <<'PY'
import json,sys
m=json.load(open(sys.argv[1]))
for d in [m["config"]["digest"]]+[l["digest"] for l in m["layers"]]:
    print("blobs/"+d.replace(":","-"))
PY
} > /tmp/qwen_files.txt
# 5.9GB GGUF 는 거의 압축이 안 되므로 gzip -1 로 빠르게(파일명은 .tgz 유지, -xzf 로 복원).
tar -C "$MODELS" -I 'gzip -1' -cf ollama_bundle/ollama-models-qwen2.5vl.tgz -T /tmp/qwen_files.txt
echo "Ollama 모델 번들: $(du -sh ollama_bundle/ollama-models-qwen2.5vl.tgz | cut -f1)"
```

> qwen2.5vl:7b ≈ 6GB(blob 6개, 단일 layer 5.9GB). 4090 24GB면 7b 권장(품질↑). 두 GPU 중 1개만 써도 충분.
> 모델 전부를 통째로 담으려면 `tar -C "$MODELS/.." -I 'gzip -1' -cf ...tgz models` 로 `models/` 전체를 묶어도 된다(다른 모델까지 포함되어 커짐).

## 7. 빌드 머신: 오프라인 번들 구성 → 반입

PART 1에서 모은 것을 **한 디렉토리**(예: `ocr_offline_bundle/`)로 묶어 반입한다.

```bash
# 프로젝트 루트에서 — 코드 + 수집물(wheelhouse/models/debs/ollama_bundle)을 함께 묶는다.

  backend frontend scripts requirements \
  CLAUDE.md README.md DESIGN.md model_manifest.json \
  wheelhouse models debs ollama_bundle \
  samples
echo "번들 크기: $(du -sh ../ocr_offline_bundle.tgz | cut -f1)"
```

구성 결과(펼쳤을 때):

```
ocr_offline_bundle/
├── (코드 전체)              backend/ frontend/ scripts/ requirements/ 등 레포 전체
├── wheelhouse/             # 3단계 — 파이썬 휠 (+ pip/setuptools/wheel 부트스트랩 휠)

│   ├── easyocr/
│   └── hf/
├── debs/                   # 5단계 — python3.12-venv · fonts-nanum (+선택 poppler/tesseract)
├── ollama_bundle/          # 6단계 — ollama tarball + 모델 tgz
└── requirements/
    └── internal_full.txt   # 새 venv 설치용 단일 파일
```

> `model_manifest.json`, `samples/` 도 함께. `data/outputs/` 는 비워서 가도 됨.

### 8-0. 번들 펼치기

> ⚠️ **전제(가장 중요)**: 내부망 머신에 **`python3.12` 가 설치돼 있어야 한다.**
> `python3.12 --version` 이 `3.12.x` 를 내면 OK. 없으면(예: 시스템이 3.10뿐) 이 번들만으로는
> 못 깐다 — `python3.12` 자체의 .deb(+의존성)를 PART 1 에서 추가로 받아와야 한다.
> (맨 `python` 이 3.10 이어도, `python3.12` 만 있으면 됨.)

```bash
python3.12 --version    # → Python 3.12.x 여야 진행 가능

# 프로젝트 위치 예: /data/ocr
sudo mkdir -p /data/ocr && sudo chown "$USER" /data/ocr && cd /data/ocr
tar -xzf /media/usb/ocr_offline_bundle.tgz -C /data/ocr   # 반입 경로에 맞게
ls                                                        # backend wheelhouse models debs ...
```

### 8-1. (필요시) python3.12-venv 설치

먼저 venv 생성이 되는지 본다. 안 되면 .deb로 깐다.

```bash
python3.12 -c "import ensurepip" 2>/dev/null && echo "ensurepip OK" || echo "ensurepip 없음 → .deb 설치 필요"
# 없으면:
sudo dpkg -i debs/python3*-venv*.deb debs/python3-pip-whl*.deb 2>/dev/null || sudo apt-get -f install
```

### 8-2. OCR 전용 새 venv 생성 (★Python 3.12★)

```bash
# A) ensurepip 가 되면 한 줄로 끝
python3.12 -m venv .venv

# B) ensurepip 가 안 되면(.deb 도 못 깔았을 때): pip 없이 만들고 wheelhouse 로 부트스트랩
python3.12 -m venv --without-pip .venv
# 휠 zip 안의 pip 을 직접 실행해 pip/setuptools/wheel 을 설치 (오프라인 부트스트랩)
PIPWHL=$(ls wheelhouse/pip-*.whl | head -1)
.venv/bin/python "$PIPWHL/pip" install --no-index --find-links wheelhouse pip setuptools wheel

# 활성화 + 버전/소속 확인
source .venv/bin/activate
python --version                  # → Python 3.12.x (필수)
python -m pip --version           # → ... (python 3.12)  ← pip 이 3.12 에 속하는지 확인
which python                      # → /data/ocr/.venv/bin/python
```

> ⚠️ **맨 `python`(=3.10) 으로 venv 를 만들지 말 것.** 반드시 `python3.12 -m venv`.
> 3.10 venv 가 되면 cp312 휠이 안 맞아 설치가 깨진다. 활성화 후엔 그 venv의 `python`/`pip`
> 이 PATH 최상단이라 맨이름으로 써도 안전하다(= 그 venv = 3.12 에 묶임).

### 8-3. 시스템 패키지(.deb) 설치 (폰트 등)

```bash
sudo dpkg -i debs/fonts-nanum*.deb 2>/dev/null || sudo apt-get -f install
fc-cache -f                                          # 폰트 캐시 갱신(선택)
# poppler/tesseract 를 받았다면 함께: sudo dpkg -i debs/poppler-utils*.deb debs/tesseract*.deb
```

### 8-4. 파이썬 휠 오프라인 설치 (전부 wheelhouse 에서)

```bash
# venv 활성화 상태에서 — ★2단계★ (paddle 은 torch 와 충돌하므로 반드시 --no-deps 로 먼저)
# (a) paddlepaddle-gpu cu129 단독
pip install --no-index --find-links wheelhouse --no-deps --force-reinstall paddlepaddle-gpu==3.3.1
# (b) 나머지 전부 — torch(cu129)·OCR·numpy2·웹
pip install --no-index --find-links wheelhouse -r requirements/internal_full.txt
```

> `--no-index` 라 PyPI로 절대 안 나간다. torch/torchvision 은 constraints 핀대로 cu129 버전,
> numpy 는 2.x, OCR 엔진 코어 3종은 검증 버전(3.6.x / 4.57.6)이 깔린다. paddle 을 (a)에서
> `--no-deps` 로 먼저 깔지 않고 (b)에 합치면 torch 와 CUDA 의존성 충돌로 설치가 실패한다.

## 11. 운영 메모

- **2× GPU 활용**: 기본은 `gpu:0` 1장만 쓴다. 두 번째 GPU를 쓰려면
  `CUDA_VISIBLE_DEVICES=1 ...` 로 두 번째 워커/인스턴스를 띄우거나, Ollama와 Paddle을
  서로 다른 GPU에 분리(`CUDA_VISIBLE_DEVICES`)해 메모리 경합을 피한다.
- **문서 OCR 기본 설정(고정)**: 엔진 `paddle --table`(레이아웃 ON=PPStructureV3).
  레이아웃 OFF면 큰 이미지가 다운스케일돼 조밀 단락이 깨진다(doc_01 41.6%→1.4%).
- **VLM 백엔드 선택**(`OCR_HYBRID_VLM`): `paddle_vl`(완전 in-process offline, Ollama
  불필요) | `ollama_vl`(Qwen) | `auto`(Ollama 떠 있으면 Qwen, 아니면 paddle_vl).
  내부망에서 Ollama 운영 부담을 줄이려면 `paddle_vl` 고정도 좋은 선택(9단계 생략 가능).
- **후처리 정규화** 기본 ON(₩ 기호 복원 등). 끄려면 `OCR_NORMALIZE=off`.
- 모든 모델/휠은 번들에 포함되어 있으므로 **재배포 시 같은 번들을 복사**하면 된다
  (다운로드 단계 재실행 불필요).
